## Smart Contract Deployment

### Hedera Contract Deployment (FT Only)

This section is **only required for Fungible Token (FT)** bridging.

#### Step 1: Set Up Files in Remix IDE

1. Navigate to [Remix IDE](https://remix.ethereum.org)
2. Create new files matching the names in your project's contract folder:
   - `hederaft.sol`
   - `HederaTokenService.sol`
   - `HederaResponseCodes.sol`
   - `IHederaTokenService.sol`

#### Step 2: Compile the Contract

1. Copy the source code from your project's contract folder
2. Paste each file's content into the corresponding Remix IDE file
3. Click on the `hederaft.sol` contract
4. Compile the contract
5. Copy the generated bytecode

#### Step 3: Configure Deployment

1. Open `Token.bin` in your project's contract folder
2. Paste the bytecode you copied from Remix IDE
3. Open `deploytoken.js` in the contract folder
4. Update the following credentials:
   - Account ID
   - Private Key

#### Step 4: Make changes to Pckage.json

Remove this line from package.json from your project source code

"type": "module"

when you are going to deploy the contract by running the below command 
and if you are not deploying the contract then keep this as it is

#### Step 5: Deploy the Contract

Run the following command in your VS Code terminal:

```bash
node src/contract/deploytoken.js
```

After successful deployment, you will receive:
- Contract ID
- EVM Address

**Important:** Update these values in the `bridgetoken.jsx` page.

### Solana Contract Deployment

#### Step 1: Prepare Your Contract

1. Copy `sol.rs` from your project's contract folder
2. Navigate to [Solana Playground](https://beta.solpg.io/)
3. Connect your wallet or generate a new one
4. Send some SOL to your wallet for deployment fees

#### Step 2: Set Up the Project

1. Create a new project in Solana Playground
2. Choose **Native (Rust)** as the project type
3. Click **Create**

#### Step 3: Deploy the Contract

1. Inside `src/lib.rs`, paste your `sol.rs` code
   - **Important:** Do not change any file names
2. Click **Build**
3. Click **Deploy**

#### Step 4: Update Configuration

After successful deployment:
1. Copy the Program ID (contract address) from line 15
2. Paste it into the `bridgetoken.jsx` page

## Supabase Database Setup

You need to create multiple tables and obtain your API keys.

### NFT Bridge Tables

#### Create Required Tables

1. Open your Supabase project
2. Navigate to the **SQL Editor**
3. Copy and paste the following SQL command:

```sql
CREATE TABLE nft_collections (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    solana_collection_mint VARCHAR(44) NOT NULL,
    hedera_token_id VARCHAR(20),
    solana_wallet VARCHAR(44),
    hedera_account_id VARCHAR(15),
    bridge_direction VARCHAR(20) CHECK (bridge_direction IN ('solanaToHedera', 'hederaToSolana')),
    initial_chain VARCHAR(10) CHECK (initial_chain IN ('solana', 'hedera')),
    preminted_hedera_nfts JSONB DEFAULT '[]'::jsonb,
    preminted_solana_nfts JSONB DEFAULT '[]'::jsonb,
    bridge_counter INTEGER DEFAULT 0,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE TABLE bridged_nfts (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    solana_nft_mints JSONB NOT NULL,
    hedera_serial_numbers JSONB NOT NULL,
    hedera_token_id VARCHAR(20),
    solana_collection_mint VARCHAR(44),
    solana_wallet VARCHAR(44),
    hedera_account_id VARCHAR(15),
    initial_chain VARCHAR(10) CHECK (initial_chain IN ('solana', 'hedera')),
    current_chain VARCHAR(10) CHECK (current_chain IN ('solana', 'hedera')),
    bridged_back BOOLEAN DEFAULT FALSE,
    nft_count INTEGER DEFAULT 1,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    last_updated TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Enable RLS but create permissive policies
ALTER TABLE nft_collections ENABLE ROW LEVEL SECURITY;
ALTER TABLE bridged_nfts ENABLE ROW LEVEL SECURITY;

-- Allow all operations for everyone on nft_collections
CREATE POLICY "Open access for all operations" ON nft_collections
    FOR ALL USING (true);

-- Allow all operations for everyone on bridged_nfts
CREATE POLICY "Open access for all operations" ON bridged_nfts
    FOR ALL USING (true);
```

4. Click **Run**

This will create both `nft_collections` and `bridged_nfts` tables with appropriate security policies.

### Fungible Token Bridge Tables

#### Create Required Tables

1. In the Supabase **SQL Editor**, copy and paste the following SQL command:

```sql
-- Create the main bridge registry table for fungible tokens
CREATE TABLE fungible_token_bridge (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    hedera_token_id VARCHAR(20) NOT NULL,
    solana_mint_address VARCHAR(44) NOT NULL,
    bridge_direction VARCHAR(20) NOT NULL CHECK (bridge_direction IN ('hederaToSolana', 'solanaToHedera')),
    hedera_creator VARCHAR(20),
    solana_creator VARCHAR(44),
    token_name VARCHAR(100) NOT NULL,
    token_symbol VARCHAR(20) NOT NULL,
    decimals INTEGER NOT NULL DEFAULT 8,
    initial_chain VARCHAR(10) NOT NULL CHECK (initial_chain IN ('hedera', 'solana')),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Create bridge transactions table
CREATE TABLE bridge_transactions (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    bridge_record_id UUID NOT NULL REFERENCES fungible_token_bridge(id) ON DELETE CASCADE,
    user_hedera_account VARCHAR(20),
    user_solana_wallet VARCHAR(44),
    direction VARCHAR(20) NOT NULL CHECK (direction IN ('hederaToSolana', 'solanaToHedera')),
    amount DECIMAL(30, 8) NOT NULL,
    token_symbol VARCHAR(20),
    hedera_transaction_id VARCHAR(100),
    solana_transaction_signature VARCHAR(100),
    status VARCHAR(20) NOT NULL DEFAULT 'pending' CHECK (status IN ('pending', 'completed', 'failed', 'cancelled')),
    bridge_step VARCHAR(50) NOT NULL CHECK (bridge_step IN ('locked_and_minted', 'transferred_and_released', 'burned_and_released')),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    completed_at TIMESTAMP WITH TIME ZONE
);

-- Enable RLS
ALTER TABLE fungible_token_bridge ENABLE ROW LEVEL SECURITY;
ALTER TABLE bridge_transactions ENABLE ROW LEVEL SECURITY;

-- Allow anyone to read bridge registry
CREATE POLICY "Allow read access to all bridge tokens" ON fungible_token_bridge
    FOR SELECT USING (true);

CREATE POLICY "Allow read access to all bridge tokens" ON bridge_transactions
    FOR SELECT USING (true);
```

2. Click **Run**

This will create the `fungible_token_bridge` and `bridge_transactions` tables with appropriate security policies.

### Obtaining Supabase API Credentials

1. Click on **Project Settings** in the left sidebar
2. Navigate to **API** section
3. Copy the **Project URL**
4. Click on **API Keys**
5. Select **anon / public** key
6. Copy the **secret_role (secret) key**

**Important:** Update these credentials in both:
- `App.jsx`
- `bridgetoken.jsx`

Replace the placeholder for "Solana Program ID" with your Supabase credentials.

## Configuration Requirements

### Network Configuration

**Important Network Settings:**

- All Mirror Node APIs are configured for **testnet** by default
- When deploying to **mainnet**, you must update:
  - All Mirror Node API URLs
  - Helius API URLs
  - Change all instances of `devnet` to `mainnet`

### API Keys Required

You need to obtain and configure the following API keys:

#### 1. Pinata API Credentials
Update in the following files:
- `api/decode-hcs.js`
- `api/upload-image.js`

Required keys:
- Pinata API Key
- Pinata Secret Key

#### 2. Helius API Key
Update wherever Helius API is referenced in your codebase.

## Platform Deployment

### Deploying to Vercel

#### Step 1: Prepare Your Repository

1. Create a new GitHub repository
2. Push your source code to the repository

#### Step 2: Deploy on Vercel

1. Create an account on [Vercel](https://vercel.com)
2. Click **"Add New"**
3. Select **"Project"**
4. Choose your GitHub repository
5. Click **"Deploy"**

Vercel will automatically build and deploy your application.
